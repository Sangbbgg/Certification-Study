# 문제 풀이 (동적 렌더링 및 실시간 채점)

다양한 문제 유형(객관식, 주관식, 찾아넣기)에 맞게 화면을 동적으로 구성하고, 사용자의 답안을 처리하여 실시간 피드백을 제공하는 기능을 담당합니다.

## 1. 개요
- **역할:** 인터랙티브 UI 구현 및 클라이언트 사이드 채점 로직.
- **주요 흐름:** 탭 진입 -> 캐시 데이터(allQuestions) 로드 -> 유형별 UI 생성 -> 답안 제출 -> 결과 및 해설 표시.

## 2. 주요 기능 구현 지침
### 2.1. 유형별 동적 UI
- **MULTIPLE_CHOICE:** `options` 기반 라디오 버튼 리스트.
- **SHORT_ANSWER:** 단일 텍스트 입력창 (`trim` 처리 필수).
- **FIND_AND_FILL:** `answer` 개수 만큼 번호가 달린 그리드 형태의 입력 영역 생성.

### 2.2. 채점 및 피드백
- **힌트 보기:** 해설 영역을 노출하되 정답은 숨겨 자가 학습 유도.
- **결과 수신:** `analyticsController.submitAnswer` 호출 후 정답 여부에 따라 결과 화면 풀스크린 렌더링.
- **이미지 렌더링:** `marked.js`를 사용하고 드라이브 호스팅 URL 구조(`lh3.googleusercontent.com/d/ID`) 연동.

### 2.3. 필터 및 정렬
- **필터 동기화:** 문제 관리와 동일한 계단식 필터 인터랙션 적용.
- **정렬 옵션:** 오래된순(기본), 최근순, 정답률 저조순 제공.

## 3. 제약사항
- **즉시성:** 필터 조작 시 별도 적용 버튼 없이 즉시 문제가 새로 로드되어야 합니다.
- **보안:** 힌트 보기 시 클라이언트 측에서 정답을 미리 알 수 없도록 데이터 바인딩을 신중히 처리합니다.
