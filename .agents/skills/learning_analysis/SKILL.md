# 학습 분석 (CRUD 및 데이터 분석)

등록된 문제 목록을 관리하고 학습 이 데이터를 바탕으로 정답률 등 분석 지표를 계산하여 시각화하는 기능을 담당합니다.

## 1. 개요
- **역할:** 프론트엔드 UI/UX 설계, 백엔드 CRUD 로직 및 데이터 분석.
- **주요 흐름:** 목록 조회(캐시 활용) -> 상세 모달 -> 수정/삭제 -> 전역 상태 동기화.

## 2. 주요 기능 구현 지침
### 2.1. 문제 목록 및 필터링
- **계단식 필터 (Cascading):** 대분류 선택 시 하위 소분류만 노출되도록 동적 필터링 처리.
- **실시간 검색:** 별도의 버튼 없이 필터 변경 즉시 캐시된 `allQuestions` 데이터를 새로 렌더링.
- **분석 지표 노출:** 각 문제 항목에 최종 풀이 시간 및 누적 정답률(%) 표시.

### 2.2. 수정 및 삭제 로직
- **지문/이미지 분리:** 지문 입력란에는 텍스트만 표시하고, 이미지는 별도 미리보기 필드 및 위치(TOP/BOTTOM) 설정 옵션 제공.
- **이미지 교체:** 이미지 변경 시 기존 드라이브 파일 삭제 후 신규 파일을 업로드하여 저장소 효율 관리.
- **데이터 정합성:** 수정/삭제 성공 후 즉시 `loadGlobalData`를 호출하여 전역 캐시 무효화 및 리로드.

## 3. 데이터베이스 (learning_history)
- 학습 이력을 저장하는 별도 테이블을 참조하여 `getQuestionsWithAnalytics` 함수에서 JOIN 또는 집계 쿼리 수행.

## 4. 제약사항
- 이미지 삭제 시 파일 부재 에러가 전체 로직을 중단시키지 않게 `try-catch` 처리합니다.
- 0으로 나누기(분모 가 0인 경우) 등 통계 계산 예외 처리를 철저히 합니다.
