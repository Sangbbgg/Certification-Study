# [스킬 01] 문제 관리 (문제 작성, 이미지 드라이브 업로드 및 조회) 로직

## 1. 역할 (Role)
* 너는 백엔드 API, 데이터베이스 스키마 설계 및 파일 시스템 연동 전문가야.

## 2. 목표 (Objective)
* Supabase에 자격증 학습용 문제 테이블(`questions`) 스키마를 설계하고, GAS 라우터를 통해 문제 등록 시 첨부된 이미지를 구글 드라이브에 업로드한 뒤, 그 URL을 마크다운 지문에 포함하여 DB에 저장하는 로직을 작성해.

## 3. 기술 스택 및 연동 흐름 (Tech Stack & Flow)
* **사용 기술:** Supabase, GAS (`supabaseFetch` 유틸리티, `DriveApp`)
* **흐름:** 클라이언트 폼 입력 (이미지 파일 포함) -> GAS 라우터(`action=createQuestion`) -> GAS `DriveApp`으로 이미지 업로드 및 공개 URL 획득 -> 지문(`content`) 마크다운에 URL 삽입 -> Supabase REST API (INSERT)

## 4. 세부 요구사항 (Requirements)
* **데이터베이스 스키마 설계 (`questions` 테이블):**
  * `id`: UUID (Primary Key, 자동 생성)
  * `major_subject`: Text (대과목)
  * `minor_subject`: Text (소과목)
  * `question_title`: Text (문제명)
  * `content`: Text (지문 - 마크다운 지원, 설명/코드/구글 드라이브 이미지 URL 포함)
  * `hint_explanation`: Text (힌트 및 해설 - 마크다운 지원)
  * `question_type`: Text (문제 유형: 'MULTIPLE_CHOICE'(객관식), 'SHORT_ANSWER'(단답/서술형), 'FIND_AND_FILL'(보기에서 찾아넣기))
  * `options`: JSONB 
    * 객관식('MULTIPLE_CHOICE'): 사용자가 동적으로 추가한 여러 개의 보기 리스트 (예: `["설계", "구현", "테스트"]`)
    * 보기에서 찾아넣기('FIND_AND_FILL'): 빈칸(구역)별로 사용자에게 보여줄 선택지(또는 매핑 정보) (필요시 사용)
    * 주관식('SHORT_ANSWER'): `null`
  * `answer`: JSONB
    * 객관식: 사용자가 동적 보기 중에서 '정답'으로 체크한 항목의 인덱스 또는 텍스트 배열
    * 보기에서 찾아넣기: 사용자가 지정한 빈칸(구역) 수만큼 직접 작성한 정답 값 배열
    * 단답형: 단일 정답 또는 허용 정답 배열
  * `created_at`: Timestamp (자동 생성)

* **구글 드라이브 이미지 업로드 함수 (`uploadImageToDrive`):**
  * 클라이언트에서 전달된 Base64 이미지 데이터를 디코딩하여 구글 드라이브에 파일로 생성.
  * 중복 파일명 방지를 위해 **GAS 백엔드(`createQuestion`)에서 `Utilities.getUuid()`를 이용해 문제 ID를 사전에 생성하고, 이 UUID를 파일명으로 사용하여 저장**할 것.
  * 프론트엔드 마크다운에서 렌더링되어야 하므로, 생성된 파일의 권한을 `DriveApp.Access.ANYONE_WITH_LINK`로 설정하여 누구나 볼 수 있는 URL을 반환할 것.

* **GAS 백엔드 처리 로직:**
  * **`createQuestion(payload)`**:
    1. `Utilities.getUuid()`를 통해 새로운 문제의 ID를 생성 (`payload`의 이미지 파일명 및 Supabase INSERT 시 `id` 값으로 사용).
    2. 생성된 UUID를 기반으로 이미지를 드라이브에 업로드하고 Markdown 지문에 병합.
    3. 사전 생성된 UUID를 포함하여 Supabase REST API 호출 (INSERT).
  * DB에서 문제 목록을 조회하는 `getQuestions(filters)` 함수 작성.
* **클라이언트 데이터 최신화 (Cache Invalidation):**
  * 문제 생성, 수정, 삭제 성공 시 반드시 `loadGlobalData(() => { callback(); }, true)`를 호출하여 전역 `allQuestions` 변수를 최신화하고 화면을 리로드한다.

## 5. 제약사항 및 예외처리 (Constraints)
* 이미지 업로드용 드라이브 폴더 ID는 절대 하드코딩하지 말고 `getEnv('IMAGE_FOLDER_ID')`를 통해 환경변수에서 가져올 것.
* 필수 값 누락 또는 파일 업로드 실패 시, DB 통신 전에 GAS 단에서 에러를 발생시키고 `sheetLogger`로 시트에 로깅한 뒤 클라이언트에게 400/500 에러를 반환할 것.

## 6. 출력 형태 (Output Format)
* Supabase SQL 에디터에서 실행할 수 있는 테이블 생성 DDL 스크립트.
* 드라이브 업로드 로직과 백엔드 컨트롤러(`questionController.gs` 파일 용) 코드가 포함된 옵시디언 마크다운 코드 블록.
