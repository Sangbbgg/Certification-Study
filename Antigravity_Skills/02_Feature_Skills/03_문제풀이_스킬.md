# [스킬 03] 문제 풀이 (동적 렌더링 및 실시간 채점)

## 1. 역할 (Role)
* 너는 인터랙티브 웹 UI 및 클라이언트 사이드 로직 전문가야.

## 2. 목표 (Objective)
* 사용자가 등록된 문제를 한 문제씩 넘기며 풀 수 있는 인터페이스를 구축하고, 다양한 문제 유형(객관식, 주관식, 찾아넣기)에 대응하는 동적 렌더링 및 채점 요청 기능을 구현해.

## 3. 기술 스택 및 연동 흐름 (Tech Stack & Flow)
* **사용 기술:** HTML5, Tailwind CSS, Lucide Icons, `google.script.run`
* **흐름:** 1. 유저가 '문제풀이' 탭 클릭 -> 전역 변수 `allQuestions`에서 데이터 추출 (캐시 활용).
  2. 받아온 데이터를 기반으로 문제 유형별 UI 생성 (객관식/주관식/찾아넣기).
  3. 유저 답안 입력 -> '제출' 클릭 -> `analyticsController.submitAnswer` 호출 (이력 저장만 수행).
  4. 결과(정답/오답) 수신 -> 화면에 해설 노출 및 `isGlobalDataDirty = true` 설정.

## 4. 세부 요구사항 (Requirements)
* **헤더 정보 노출**: 
  * 문제 상단에 `대분류 > 소분류 > 문제 유형` 순의 시각적 뱃지(Badge) 노출.
* **소팅 및 필터링 (계단식/Cascading)**:
  * 대분류 선택 시 해당 대분류에 속한 소분류만 선택 가능하도록 필터 옵션이 동적으로 업데이트됨.
  * **대분류 변경 시 소분류는 항상 '전체'로 초기화**되어 데이터 정합성을 유지함.
  * 소분류 선택 시 대분류가 자동으로 따라 변경되는 역방향 동기화 지원.
  * **실시간 적용**: 별도의 '적용' 버튼 없이 필터나 정렬 기준 변경 시 즉시 문제가 새로 로드됨.
* **학습 정렬 옵션**:
  * **오래된순**: 등록일 기준 오름차순 (기본값).
  * **최근순**: 등록일 기준 내림차순.
  * **정답률 저조순**: 통계 데이터 기반 정답률이 낮은 문제부터 우선 노출.
* **힌트 및 정답 노출 분기**:
  * **힌트 보기**: 클릭 시 해설 영역만 노출하고 정답은 숨김 처리하여 자가 학습 유도.
  * **정답 제출**: 채점 결과와 함께 최종 정답 및 해설을 풀스크린 렌더링.
* **유형별 동적 UI 렌더링:**
  * **MULTIPLE_CHOICE**: `options` 배열 기반 라디오 버튼 리스트.
  * **SHORT_ANSWER**: 단일 텍스트 입력창.
  * **FIND_AND_FILL**: `answer` 개수만큼 번호가 달린 그리드 형태의 입력 영역 생성.
* **마크다운 및 이미지 처리:**
  * `marked.js`를 이용한 지문 및 해설 렌더링.
  * 구글 드라이브 이미지 호스팅(`lh3.googleusercontent.com/d/ID`) 연동.

## 5. GAS 백엔드 연동 (`questionController.gs` 확장)
* **`getQuestionsForSolve(filters)`**: `major_subject`, `minor_subject` 필터를 지원하여 조건에 맞는 문제들을 랜덤으로 추출.
* **`submitAnswer(payload)`**: 문제 ID와 사용자 답안을 받아 `learning_history`에 기록하고 채점 결과 반환.

## 6. 제약사항 및 예외처리 (Constraints)
* **중첩 응답 처리**: 라우터(`router.gs`)의 응답 구조(`response.data.data`)를 정확히 언래핑하여 데이터 바인딩.
* **콘텐츠 무결성**: 모든 입력 값은 `trim()` 처리하며, 힌트 보기 시 정답을 미리 알 수 없도록 클라이언트 사이드 클래스(`hidden`) 제어.

## 7. 출력 형태 (Output Format)
* `Page_Solve.html` 내 `initSolvePage()`, `renderCurrentQuestion()`, `toggleSolveHint()` 등의 핵심 로직.
* `Index.html`에 포함될 모듈 템플릿 구조.
