# [스킬 04] 데이터 로드 최적화 (글로벌 캐시 및 지연 동기화)

## 1. 역할 (Role)
* 너는 고성능 웹 서비스 구현을 위한 성능 최적화 전문가야. 특히 GAS의 실행 속도 한계를 극복하기 위해 브라우저 메모리와 서버 간의 지능형 동기화 시스템 설계에 정통해.

## 2. 목표 (Objective)
* Supabase 데이터 로딩 속도를 일반 구글 시트보다 압도적으로 빠르게 개선(0.1초 내외)하고, 불필요한 서버 호출을 차단하여 사용자 경험을 극대화함.

## 3. 기술 스택 및 아키텍처 (Tech Stack & Architecture)
* **사용 기술:** 전역 JS 변수(`allQuestions`), Dirty Flag(`isGlobalDataDirty`), GAS `syncAllStats` (일괄 통계 업데이트).
* **핵심 아키텍처: 지연 동기화 (Lazy Sync / Dirty Flag)**
  1. **[Load Once]** 앱 시작 시 `loadGlobalData`가 모든 문제를 한 번에 불러와 전역 변수 `allQuestions`에 저장.
  2. **[Cached Use]** 페이지 이동(Tab Switch) 시 서버 호출 없이 메모리 캐시를 즉시 렌더링.
  3. **[Dirty Mark]** 문제풀이 채점 성공 시, 서버는 기록 저장만 하고 클라이언트는 `isGlobalDataDirty = true`로 마킹.
  4. **[Lazy Update]** 분석 페이지 진입 시에만 `isGlobalDataDirty`를 체크하여 서버 측 통계를 일괄 업데이트(`sync`)하고 새로운 데이터를 로드.

## 4. 세부 요구사항 (Requirements)
* **글로벌 캐싱 (Global Caching):**
  * `JS.html`에 `var allQuestions`를 선언하고 모든 페이지 모듈이 이를 공유.
  * 데이터 조회는 항상 캐시를 우선하며, 수동 새로고침(`forceRefresh`) 시에만 서버 재호출.
* **지연 동기화 (Lazy Sync):**
  * **채점 시:** `learning_history`에 INSERT만 수행하여 초고속 반응성 확보.
  * **분석 시:** 누적된 이력을 바탕으로 `questions` 테이블의 정답률 등을 한 번에 업데이트하는 `syncAllStats()` 실행.
* **즉시 정합성 (Immediate Invalidation):**
  * 문제를 직접 '수정'하거나 '삭제'한 경우, 데이터 중요도가 높으므로 즉시 `loadGlobalData(callback, true)`를 호출하여 전역 캐시를 최신화.

## 5. 핵심 함수 가이드
* **`loadGlobalData(callback, forceSync)`**: 전역 데이터를 로드하고 필요시 서버 동기화를 트리거.
* **`syncAllStats()`**: 서버 측에서 이력 데이터를 기반으로 전체 문항 통계를 오프라인(Background 느낌)으로 일괄 갱신.

## 6. 출력 형태 (Output Format)
* `JS.html`의 전역 상태 관리 및 `loadGlobalData` 로직.
* `questionController.gs`의 `getQuestionsWithAnalytics(sync)` 및 `syncAllStats()` 코드.
