# [스킬 02] 문제관리 & 분석 (조회, 수정, 삭제 및 통계)

## 1. 역할 (Role)
- 너는 프론트엔드 UI/UX 설계, 백엔드 CRUD 로직 및 데이터 분석 전문가야.

## 2. 목표 (Objective)
- 등록된 문제 목록을 조회하고, 상세 내용을 확인 및 수정/삭제할 수 있는 관리 기능을 구현해. 또한 기존 학습 이력을 바탕으로 정답률 등 분석 데이터를 함께 노출해.

## 3. 기술 스택 및 연동 흐름 (Tech Stack & Flow)
- **사용 기술**: Supabase, GAS, DriveApp
- **흐름 (조회)**: Index.html(관리 탭) -> `isGlobalDataDirty` 체크 -> `true`일 경우 `getQuestionsWithAnalytics(sync=true)` 호출하여 실시간 통계 업데이트 후 로드 -> 결과 리스트 출력
- **흐름 (수정)**: 상세 팝업/영역 -> `updateQuestion` 호출 -> 이미지 변경 시 (기존 드라이브 파일 삭제 -> 신규 업로드) -> Supabase (UPDATE) -> 전역 캐시 즉시 리로드

## 4. 세부 요구사항 (Requirements)

- `created_at`: Timestamp (자동 생성)

#### `questions` 테이블 (상세 명세)
- `id`: UUID (Primary Key)
- `major_subject`: TEXT (대과목)
- `minor_subject`: TEXT (소과목)
- `question_title`: TEXT (문제명)
- `content`: TEXT (지문 텍스트 - 이미지 태그 제외)
- `image_url`: TEXT (이미지 경로 - 외부 호스팅/드라이브 URL)
- `image_position`: TEXT (이미지 표시 위치: TOP/BOTTOM)
- `question_type`: TEXT ('MULTIPLE_CHOICE', 'SHORT_ANSWER', 'FIND_AND_FILL')
- `options`: JSONB (객관식 보기 항목 리스트)
- `answer`: JSONB (정답 데이터: 객관식은 인덱스/값, 단답형은 텍스트)
- `hint_explanation`: TEXT (해설 마크다운)
- `created_at`: TIMESTAMP (생성일자)

### 4.2. 화면 구성 (UI)
- **문제 목록 (List View)**:
  - **소팅/필터링 (계단식/Cascading)**: 
    - 문제풀이 페이지와 동일한 인터랙션을 공유함.
    - **대분류 선택 시**: 해당 대분류에 속한 소분류만 선택 가능하도록 소분류 필터 옵션이 실시간으로 필터링되며, **기존 선택된 소분류는 '전체'로 초기화**됨 (`onMajorFilterChange`).
    - **소분류 선택 시**: 해당 소분류에 맞는 대분류가 자동으로 상단 필터에 따라 변경되는 역방향 동기화 지원 (`onMinorFilterChange`).
    - 모든 필터 조작 시 목록이 즉각적으로 갱신되어 캐시된 `allQuestions` 데이터를 즉시 렌더링함.
  - 컬럼: 문제명, 최종 풀이 시간, 정답률(%), 관리(수정/삭제 버튼)
  - 문제명 클릭 시 상세 보기 모달/영역 활성화
- **상세 및 수정 (Detail & Edit)**:
  - 초기값은 조회 모드, '수정 토글' 클릭 시 입력 폼으로 전환
  - **지문/이미지 분리**: 지문 입력란에는 텍스트만 표시하고, 이미지는 별도 미리보기 및 업로드 필드로 관리.
  - **위치 제어**: 이미지 표시 위치(지문 위/아래)를 선택할 수 있는 옵션 제공.
  - **정답 및 보기 수정**: 
    - 문제 유형에 따라 보기(Options)를 추가/삭제하거나 내용을 수정할 수 있는 동적 입력 UI 제공.
    - 정답(Answer) 값을 직접 변경할 수 있도록 지원.
  - **헤더 정보 노출**: 상세 보기 상단에 대분류, 소분류 명칭과 문제 유형을 시각적인 뱃지(Badge) 형태로 노출하여 직관성 강화 (대분류 > 소분류 > 문제 유형 순).

### 4.3. 백엔드 처리 로직 (GAS)
- **`updateQuestion(payload)`**:
  - `payload`의 `content`와 `image_url`, `image_position`을 각각 업데이트.
  - 이미지 변경 시에만 드라이브 파일 교체 로직 수행.
- **`deleteQuestion(id)`**: 지문 내 이미지 존재 시 드라이브 파일 삭제 후 DB 레코드 삭제.

## 5. 제약사항 및 예외처리 (Constraints)
- 이미지 삭제 시 파일 부재로 인한 에러가 전체 로직을 중단시키지 않도록 `try-catch` 처리.
- 통계 계산 시 데이터가 없는 경우(0으로 나누기 등)에 대한 예외 처리.

## 6. 출력 형태 (Output Format)
- `learning_history` 생성을 위한 SQL DDL.
- `getQuestionsWithAnalytics`, `updateQuestion` 등의 함수가 포함된 `analyticsController.gs` 및 `questionController.gs` 코드 블록.
